## 용어 정리

### 1) 관심도 점수

**정의**

> 0.7 × 네이버 검색량 비중
>
> - 0.3 × 구글 트렌드 지수  
>   (구글 데이터가 없으면 네이버 검색량 비중만 사용)

**왜 이렇게 정의했는가**

- 네이버는 국내 검색 시장에서 점유율이 높아서,  
  **“국내 관심도”를 반영할 때 기본 축으로 삼음.**
- 구글 트렌드는 글로벌/기술 커뮤니티 등에서의 인기도를 보완해 주기 때문에,  
  **보조 지표로 0.3 비중만 부여**해서 “왜곡 없이 보완”만 하도록 설계.
- 실제 구현 상, 구글 데이터가 없는 모델도 많기 때문에  
  **네이버 기준만으로도 비교 가능**하도록 기본 축은 네이버로 두었음.

---

### 2) 보급률 (Adoption Rate)

**정의**

> 보급률(%) =  
> (해당 모델의 월 판매량 ÷ 같은 기간 전체 시장 판매량) × 100

**왜 이렇게 정의했는가**

- 단순 “판매량 숫자”만 보면,  
  국내 전체 시장 상황(호황/불황, 전체 판매 감소 등)을 감안하기 어려움.
- 그래서 **모델 판매량을 전체 시장 규모로 나눈 비율**을 써서,
  - 시장이 커졌는지/줄었는지와 상관없이
  - **“해당 모델이 그 시점에 얼마나 점유하고 있는지”**를 비교할 수 있게 함.
- 이 값은 특정 월 기준 “해당 모델의 시장 내 존재감”을 보여주는 지표로 사용.

### 3) 다나와 인기순

**정의**

> 다나와 자동차에서 제공하는 “월간 인기순” 랭킹

**왜 그대로 쓰는가**

- 다나와 내부 알고리즘(조회수, 관심도, 견적 요청 등)이 복합적으로 반영된 지표라  
  **외부에서 역산하거나 재정의하기 어려움.**
- 대신, 우리 프로젝트에서는
  - **“참고용 랭킹 지표”**로만 사용하고
  - 직접적인 점수 계산(관심도/보급률)에는 포함하지 않음.
- 즉, “다나와 기준에서 사람들이 얼마나 보고 있는가”를  
  **부가적인 뷰**로만 제공.

### 4) 블로그 여론 데이터

**정의**

> 네이버 검색 결과 상위 3개 블로그 글의 본문을 수집한 뒤,  
> 명사 기반 NLP 토큰화 → 키워드 빈도 계산 → 워드클라우드로 시각화한 것.

**왜 이렇게 정의했는가**

- 실제 구매/시승/사용 후기에서 어떤 단어가 많이 등장하는지 보고 싶었음.
- 하지만
  - 모든 블로그 글을 다 수집할 수도 없고,
  - API 호출/크롤링 비용도 크기 때문에
  - **“검색 상위 3개”로 범위를 제한**하여 현실적인 선에서 구현.
- 명사 중심으로 키워드를 뽑는 이유는,
  - 고유명사, 부품명, 감정 키워드(예: 승차감, 소음, 공간감 등)를  
    **직관적으로 보이게 하기 위한 설계.**
- 이 결과를 **워드클라우드 + 키워드 테이블**로 함께 보여주어  
  “수치 데이터(판매/관심도)”와 “텍스트 여론”을 동시에 해석할 수 있게 함.
